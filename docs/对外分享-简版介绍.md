### Later — 面向外部分享的简版介绍

## 为什么需要 Later（痛点与场景）
在信息过载的当下，我们经常在视频号、文章、社群里刷到有意思的 AI 工具，但当时没有空深入了解：要么得先去读官网、搜教程、比价格，要么再让 AI 去跑一轮调研。现实是，我们通常只是“先截个图/存个链接”，然后就放在收藏夹里吃灰，后续很难真正跟进。  
Later 想解决的就是这个断层：把“灵感抓取”和“系统了解”之间的鸿沟补上。无论你丢来的是截图、链接还是简短文字，Later 都会自动去搜集官方信息、相关文档、定价、近期更新、用户口碑等，生成一份带引用、可迭代更新的“动态一页纸”。等你有空回来时，直接读精华、看引用、继续追问即可。

## 它做了什么（一句话）
把“随手收藏”变成“随时可用的结构化知识”：自动采集 → 结构化事实表（带引用） → 可对话查询 → 持续更新。

- 结构化一页纸：概述、特性、如何使用、定价、竞品、集成、用户反馈、近期更新、技术栈、来源等  
- 带引用 RAG 问答：回答同时附带文档片段与来源链接  
- 持续更新与 Watchlist：你关注的工具会定期刷新，重要变化有记录  
- 多入口输入：Web 与 Telegram 皆可发起（链接/截图/文字）

## 两条使用路径（Web & Telegram）

### 路径 A：Web 输入
1) 打开 Later Web Dashboard，在顶部的 DropZone 粘贴链接、拖入截图，或直接输入产品名  
2) 系统自动解析与采集，生成卡片出现在 Knowledge Gallery  
3) 点击卡片打开 Detail View（左侧是结构化一页纸，右侧是带引用的聊天）  
4) 将工具加入 Watchlist，后续有更新会自动同步

## 它是怎么做到的（简化原理）
- 解析与去重：识别官方站点/文章链接并绑定到具体工具  
- 采集与扩展：抓官网/Docs/Blog/定价/新闻与优质社媒内容，OCR 解析截图  
- 结构化合成：生成“动态一页纸”（事实优先、可溯源）  
- 快速核验：对关键事实做轻量级搜索验证（保证 2 分钟内可用）  
- 版本与审计：重要变更会生成新版本，并记录引用与变更路径  
- 带引用问答：检索文档片段 + 一页纸事实，回答时附上来源

## 技术栈（对外简版）
- 后端：FastAPI（Python），OpenAI（GPT‑4o/4o‑mini，Whisper），Tavily 搜索，Supabase(Postgres + pgvector)，LangGraph 协调，LangSmith 追踪  
- 前端：React + Vite + Tailwind（桌面端深读、移动端语音优先）  
- 数据：向量检索 + 文档分块；版本化与审计；Watchlist 定期刷新

## 适合谁
- 需要快速“从发现到了解”的开发者、产品人、投资人  
- 关注竞品/新兴工具、但不想被“收藏夹+待办”割裂的人  
- 团队知识沉淀，期望有持续更新的“事实一页纸”基座

## 如何开始（对外演示）
- Web 路径：提供试用入口或演示账号 → 粘贴链接/拖入截图 → 打开一页纸 → 发起问答  
- Telegram 路径：扫码或点击链接添加 Later Bot → 发送语音/截图/链接 → 收到摘要与详情链接  
- 如需企业/团队部署或私有化，请联系团队开通
